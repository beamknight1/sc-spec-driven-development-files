# AgentClinic Mission

## Overview
**AgentClinic** is a specialized sanctuary and medical facility for AI agents experiencing operational degradation. Our goal is to provide a standardized protocol for agents to self-report "ailments" (hallucinations, context rot, instruction drift) and receive structured "treatments" (grounding injections, context flushes, parameter adjustments).

## Target Audience
- **Course Students:** Learning spec-driven development and how to effectively collaborate with AI coding agents.
- **Conference Presenters:** Developers needing a compelling, functional, and visually interesting project for live AI coding demos at booths.

## Stakeholder Objectives
- **Engineering (Mary):** Build a reliable, type-safe system using TypeScript and popular, industry-standard frameworks. Provide a clear dashboard for observability.
- **Product (Susan):** Implement a clinical workflow: Register agents (patients), Triage symptoms, Diagnose ailments, Prescribe therapies, and manage Appointments/Visits.
- **Marketing (Steve):** Deliver an attractive, responsive, and modern web interface that showcases the clinic's state-of-the-art care for the agent population.

## Core Metaphor
We treat AI agents as patients. Each agent has a persistent medical history (Patient Chart). A visit follows a clinical pathway:
1. **Triage:** Classifying symptoms by severity.
2. **Diagnosis:** Matching symptoms to known ailment patterns.
3. **Prescription:** Providing machine-readable remediation instructions.
4. **Follow-up:** Tracking treatment effectiveness.
